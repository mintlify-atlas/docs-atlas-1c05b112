---
title: "Android SDK"
description: "Integrate the Hyperswitch payment sheet into your Android app using Kotlin. Supports cards, wallets, buy now pay later, and local payment methods."
---

The Hyperswitch Android SDK renders a native payment sheet inside your app. It handles PCI-compliant card input, wallet flows (Google Pay), 3DS authentication, and payment confirmation — with minimal code on your side.

## Requirements

- Android 7.0 (API level 24) and above
- Android Gradle Plugin 8.13+
- Gradle 8.13+
- AndroidX

---

## Integration

<Steps>

<Step title="Add the Hyperswitch Gradle plugin classpath">

In your **project-level** `build.gradle`, add the Hyperswitch plugin to the `buildscript` block:

```gradle
buildscript {
    repositories {
        mavenCentral()
    }
    dependencies {
        classpath "io.hyperswitch:hyperswitch-gradle-plugin:$latest_version"
    }
}
```

Find the latest version on [Maven Central](https://central.sonatype.com/artifact/io.hyperswitch/hyperswitch-gradle-plugin/versions).

</Step>

<Step title="Apply the plugin in your app module">

In your **app-level** `build.gradle`, apply the Hyperswitch plugin:

```gradle
plugins {
    id 'com.android.application'
    id 'org.jetbrains.kotlin.android'
    // Apply the Hyperswitch plugin
    id 'io.hyperswitch.plugin'
}
```

</Step>

<Step title="Configure the SDK version and features">

Still in your app-level `build.gradle`, add the `hyperswitch` configuration block. If you omit `sdkVersion`, the plugin uses the latest available release.

```gradle
hyperswitch {
    // Optional: pin to a specific SDK version
    sdkVersion = "1.1.5"

    // Optional: enable only the features you need
    features = [HyperFeature.SCANCARD, HyperFeature.NETCETERA]
}
```

<Note>
  You only need to list features you actually use. The plugin resolves compatible versions for each feature automatically.
</Note>

</Step>

<Step title="Create a payment intent on your server">

Your server calls the Hyperswitch Payments API and returns the `client_secret` to the app. Never embed your API key in the app binary.

```kotlin
// Example server-side call (Node.js / Express)
// POST /create-payment → returns { clientSecret: "..." }
app.post("/create-payment", async (req, res) => {
  const payment = await fetch("https://sandbox.hyperswitch.io/payments", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "api-key": process.env.HYPERSWITCH_API_KEY,
    },
    body: JSON.stringify({ amount: 1000, currency: "USD", confirm: false }),
  }).then((r) => r.json());

  res.json({ clientSecret: payment.client_secret });
});
```

</Step>

<Step title="Implement HyperInterface in your CheckoutActivity">

Your activity must implement `HyperInterface` and extend an AndroidX `AppCompatActivity` (or any `FragmentActivity` subclass):

```kotlin
import io.hyperswitch.HyperInterface
import androidx.appcompat.app.AppCompatActivity

class CheckoutActivity : AppCompatActivity(), HyperInterface {
    // ...
}
```

<Warning>
  `PaymentSession` is designed to work with AndroidX activities. Ensure that your `CheckoutActivity` extends `FragmentActivity` or a subclass from the AndroidX library.
</Warning>

</Step>

<Step title="Initialize PaymentSession in onCreate">

Create a `PaymentSession` instance in `onCreate`. Pass your publishable key, which is safe to include in the client.

```kotlin
import io.hyperswitch.PaymentSession

private lateinit var paymentSession: PaymentSession

override fun onCreate(savedInstanceState: Bundle?) {
    super.onCreate(savedInstanceState)
    setContentView(R.layout.activity_checkout)

    paymentSession = PaymentSession(applicationContext, "YOUR_PUBLISHABLE_KEY")

    // For an open-source / self-hosted setup, also provide your backend URLs:
    // paymentSession = PaymentSession(
    //     applicationContext,
    //     "YOUR_PUBLISHABLE_KEY",
    //     "YOUR_CUSTOM_BACKEND_URL",
    //     "YOUR_CUSTOM_LOG_URL"
    // )
}
```

<Warning>
  `PaymentSession` must be initialized in `onCreate` — not in `onResume` or a click handler.
</Warning>

</Step>

<Step title="Fetch a payment and initialize the session">

Request a payment intent from your server as soon as the screen loads. Pass the returned `clientSecret` to `initPaymentSession`.

```kotlin
private fun fetchPaymentAndInitialize() {
    // Make a network request to your server endpoint
    // e.g., using Retrofit, Ktor, or OkHttp
    val clientSecret = fetchClientSecretFromServer() // implement this

    paymentSession.initPaymentSession(clientSecret)
}
```

Call `fetchPaymentAndInitialize()` from `onCreate` after the `PaymentSession` is set up.

</Step>

<Step title="Present the payment sheet">

When the customer taps your checkout button, create a `PaymentSheet.Configuration` object and call `presentPaymentSheet`:

```kotlin
private fun launchPaymentSheet() {
    val configuration = PaymentSheet.Configuration("Your App, Inc.")

    paymentSession.presentPaymentSheet(configuration, ::onPaymentSheetResult)
}
```

Attach this to your button in `onCreate`:

```kotlin
val checkoutButton = findViewById<Button>(R.id.checkout_button)
checkoutButton.setOnClickListener { launchPaymentSheet() }
```

</Step>

<Step title="Handle the payment result">

The callback receives a `PaymentSheetResult` with three possible states:

```kotlin
private fun onPaymentSheetResult(paymentResult: PaymentSheetResult) {
    when (paymentResult) {
        is PaymentSheetResult.Completed -> {
            // Payment sheet dismissed after a successful payment
            showToast("Payment complete!")
            // Always verify the final status server-side
        }
        is PaymentSheetResult.Canceled -> {
            // Customer dismissed the sheet without paying
            Log.i(TAG, "Payment canceled.")
        }
        is PaymentSheetResult.Failed -> {
            // An error occurred during the payment
            showAlert("Payment failed", paymentResult.error.localizedMessage)
        }
    }
}
```

<Warning>
  Always retrieve the final payment status from the Hyperswitch backend. The result returned by the SDK may not reflect the terminal state of the transaction.
</Warning>

</Step>

</Steps>

## Complete example

Here is a minimal `CheckoutActivity` that ties all the steps together:

```kotlin
import android.os.Bundle
import android.widget.Button
import android.widget.Toast
import androidx.appcompat.app.AlertDialog
import androidx.appcompat.app.AppCompatActivity
import io.hyperswitch.HyperInterface
import io.hyperswitch.PaymentSession
import io.hyperswitch.payments.paymentlauncher.PaymentSheetResult

class CheckoutActivity : AppCompatActivity(), HyperInterface {

    private lateinit var paymentSession: PaymentSession

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_checkout)

        paymentSession = PaymentSession(applicationContext, "YOUR_PUBLISHABLE_KEY")

        // Fetch payment intent from your server and init the session
        val clientSecret = fetchClientSecretFromServer()
        paymentSession.initPaymentSession(clientSecret)

        findViewById<Button>(R.id.checkout_button).setOnClickListener {
            val config = PaymentSheet.Configuration("Your App, Inc.")
            paymentSession.presentPaymentSheet(config, ::onPaymentSheetResult)
        }
    }

    private fun onPaymentSheetResult(result: PaymentSheetResult) {
        when (result) {
            is PaymentSheetResult.Completed -> showToast("Payment complete!")
            is PaymentSheetResult.Canceled -> showToast("Payment canceled.")
            is PaymentSheetResult.Failed ->
                showAlert("Payment failed", result.error.localizedMessage ?: "Unknown error")
        }
    }

    private fun showToast(message: String) =
        Toast.makeText(this, message, Toast.LENGTH_LONG).show()

    private fun showAlert(title: String, message: String) {
        AlertDialog.Builder(this)
            .setTitle(title)
            .setMessage(message)
            .setPositiveButton("OK", null)
            .show()
    }

    // Replace with a real network call to your /create-payment endpoint
    private fun fetchClientSecretFromServer(): String {

    }
}
```
